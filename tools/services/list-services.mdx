## Overview

Retrieves a paginated list of all services in your PagerDuty account. You can filter services by name or team membership.

<Info>
This is a **read-only** tool. It retrieves data without modifying your PagerDuty account.
</Info>

## Parameters

<ParamField path="query_model" type="ServiceQuery" required>
  Query parameters for filtering and pagination.
  
  <Expandable title="ServiceQuery Fields">
    <ParamField path="query" type="string">
      Filters the results, showing only the services whose name matches the query string.
      
      Example: `"production"`
    </ParamField>
    
    <ParamField path="teams_ids" type="string[]">
      Filter services by team IDs. Only services associated with these teams will be returned.
      
      Example: `["PTEAM1", "PTEAM2"]`
    </ParamField>
    
    <ParamField path="limit" type="integer" default="20">
      Maximum number of results to return. Must be between 1 and 100.
      
      Default: `20`
    </ParamField>
  </Expandable>
</ParamField>

## Response

Returns a `ListResponseModel[Service]` containing an array of services.

<ResponseField name="response" type="Service[]">
  Array of Service objects matching the query parameters.
  
  <Expandable title="Service Object">
    <ResponseField name="id" type="string">
      The unique identifier for the service.
    </ResponseField>
    
    <ResponseField name="type" type="string" required>
      The type of the object (always `"service"`).
    </ResponseField>
    
    <ResponseField name="name" type="string">
      The name of the service.
    </ResponseField>
    
    <ResponseField name="description" type="string">
      The description of the service.
    </ResponseField>
    
    <ResponseField name="escalation_policy" type="EscalationPolicyReference" required>
      Reference to the escalation policy associated with this service.
      
      <Expandable title="EscalationPolicyReference">
        <ResponseField name="id" type="string" required>
          The ID of the escalation policy.
        </ResponseField>
        
        <ResponseField name="type" type="string" required>
          The type (always `"escalation_policy_reference"`).
        </ResponseField>
        
        <ResponseField name="summary" type="string">
          A short summary of the escalation policy (read-only, server-generated).
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="teams" type="TeamReference[]">
      List of teams associated with the service.
      
      <Expandable title="TeamReference">
        <ResponseField name="id" type="string" required>
          The ID of the team.
        </ResponseField>
        
        <ResponseField name="type" type="string" required>
          The type (always `"team_reference"`).
        </ResponseField>
        
        <ResponseField name="summary" type="string">
          A short summary of the team.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="response_summary" type="string">
  A human-readable summary of the response including the count and type of records returned.
</ResponseField>

## Example Usage

<CodeGroup>
```text Natural Language
List all services
```

```text With Filtering
Show me all production services
```

```text Filter by Team
List services for team PTEAM123
```
</CodeGroup>
