## Overview

Retrieves detailed information about a specific service. Use this tool when you need to examine the complete configuration and details of a particular service.

<Info>
This is a **read-only** tool. It retrieves data without modifying your PagerDuty account.
</Info>

## Parameters

<ParamField path="service_id" type="string" required>
  The ID of the service to retrieve.
  
  Example: `PSERVICE1`
</ParamField>

## Response

Returns a single `Service` object with complete details.

<ResponseField name="id" type="string">
  The unique identifier for the service.
</ResponseField>

<ResponseField name="type" type="string" required>
  The type of the object (always `"service"`).
</ResponseField>

<ResponseField name="name" type="string">
  The name of the service.
</ResponseField>

<ResponseField name="description" type="string">
  The description of the service.
</ResponseField>

<ResponseField name="escalation_policy" type="EscalationPolicyReference" required>
  Reference to the escalation policy associated with this service.
  
  <Expandable title="EscalationPolicyReference">
    <ResponseField name="id" type="string" required>
      The ID of the escalation policy.
    </ResponseField>
    
    <ResponseField name="type" type="string" required>
      The type (always `"escalation_policy_reference"`).
    </ResponseField>
    
    <ResponseField name="summary" type="string">
      A short summary of the escalation policy (read-only, server-generated).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="teams" type="TeamReference[]">
  List of teams associated with the service.
  
  <Expandable title="TeamReference">
    <ResponseField name="id" type="string" required>
      The ID of the team.
    </ResponseField>
    
    <ResponseField name="type" type="string" required>
      The type (always `"team_reference"`).
    </ResponseField>
    
    <ResponseField name="summary" type="string">
      A short summary of the team.
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Usage

<CodeGroup>
```text Natural Language
Show me details for service PSERVICE1
```

```text Get Service Info
What is the configuration for service PSERVICE1?
```
</CodeGroup>

```json
  "type": "service",
  "name": "Production Web Service",
  "description": "Main web application service handling user requests",
  "escalation_policy": {
    "id": "PEPOLICY1",
    "type": "escalation_policy_reference",
    "summary": "Production Escalation Policy"
  },
  "teams": [

      "id": "PTEAM1",
      "type": "team_reference",
      "summary": "Engineering Team"
    },
    {
      "id": "PTEAM2",
      "type": "team_reference",
      "summary": "DevOps Team"
    }
  ]
}
```
