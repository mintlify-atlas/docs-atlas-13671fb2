## Overview

Creates a new service in your PagerDuty account. Services represent the applications, components, or systems that you want to monitor.

<Warning>
This is a **write** tool that modifies your PagerDuty account. You must enable write mode with the `--enable-write-tools` flag to use this tool.
</Warning>

## Parameters

<ParamField path="service_data" type="ServiceCreate" required>
  The data for the new service.
  
  <Expandable title="ServiceCreate Fields">
    <ParamField path="service" type="Service" required>
      The service object to create.
      
      <Expandable title="Service Fields">
        <ParamField path="name" type="string" required>
          The name of the service.
          
          Example: `"Production API Service"`
        </ParamField>
        
        <ParamField path="description" type="string">
          The description of the service.
          
          Example: `"REST API service for production environment"`
        </ParamField>
        
        <ParamField path="escalation_policy" type="EscalationPolicyReference" required>
          Reference to the escalation policy for this service. Only the `id` field is required.
          
          <Expandable title="EscalationPolicyReference">
            <ParamField path="id" type="string" required>
              The ID of the escalation policy.
              
              Example: `"PEPOLICY123"`
            </ParamField>
          </Expandable>
          
          <Note>
          The `summary` field is server-generated and will be ignored if provided.
          </Note>
        </ParamField>
        
        <ParamField path="teams" type="TeamReference[]">
          List of teams to associate with the service.
          
          <Expandable title="TeamReference">
            <ParamField path="id" type="string" required>
              The ID of the team.
              
              Example: `"PTEAM1"`
            </ParamField>
          </Expandable>
        </ParamField>
      </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

<Note>
Do not include an `id` field in the service data. The service ID is automatically generated by PagerDuty.
</Note>

## Response

Returns the created `Service` object with the server-generated ID and other fields.

<ResponseField name="id" type="string">
  The unique identifier for the newly created service (server-generated).
</ResponseField>

<ResponseField name="type" type="string">
  The type of the object (always `"service"`).
</ResponseField>

<ResponseField name="name" type="string">
  The name of the service.
</ResponseField>

<ResponseField name="description" type="string">
  The description of the service.
</ResponseField>

<ResponseField name="escalation_policy" type="EscalationPolicyReference">
  Reference to the escalation policy associated with this service.
</ResponseField>

<ResponseField name="teams" type="TeamReference[]">
  List of teams associated with the service.
</ResponseField>

## Example Usage

<CodeGroup>
```text Natural Language
Create a service called "Production API Service" with escalation policy PEPOLICY123
```

```text With Teams
Create a new service named "Database Service" using escalation policy PEPOLICY456 and assign it to team PTEAM789
```
</CodeGroup>

## Example Request

```json
{
  "service_data": {
    "service": {
      "name": "Production API Service",
      "description": "REST API service for production environment",
      "escalation_policy": {
        "id": "PEPOLICY123"
      },
      "teams": [
        {
          "id": "PTEAM1"
        }
      ]
    }
  }
}
```

## Example Response

```json
{
  "id": "PSERVICE999",
  "type": "service",
  "name": "Production API Service",
  "description": "REST API service for production environment",
  "escalation_policy": {
    "id": "PEPOLICY123",
    "type": "escalation_policy_reference",
    "summary": "Production On-Call Policy"
  },
  "teams": [
    {
      "id": "PTEAM1",
      "type": "team_reference",
      "summary": "Backend Engineering"
    }
  ]
}
```

## Important Notes

<Note>
**Escalation Policy Reference**: When creating a service, the `escalation_policy` reference only requires the `id` field. The `summary` field is server-generated and will be ignored if you provide it.

Example: `{"id": "PEPOLICY123"}`
</Note>

<Warning>
Before creating a service, ensure that:
- The escalation policy ID exists and is valid
- Any team IDs you reference exist in your account
- You have the necessary permissions to create services
</Warning>
