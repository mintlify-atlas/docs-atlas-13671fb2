## Overview

The `update_alert_grouping_setting` tool modifies an existing alert grouping setting, allowing you to change how alerts are grouped into incidents for specific services.

<Warning>
This is a **write** tool that modifies an existing configuration in your PagerDuty account. You must enable write tools with the `--enable-write-tools` flag to use this tool.
</Warning>

## Parameters

<ParamField path="setting_id" type="string" required>
  The unique identifier of the alert grouping setting to update.
  
  Example: `"PAGRSET1"`
</ParamField>

<ParamField path="name" type="string" optional>
  A short-form name for labeling the alert grouping setting.
  
  Example: `"Updated Production Grouping"`
</ParamField>

<ParamField path="description" type="string" optional>
  A description providing more information about the alert grouping setting.
  
  Example: `"Updated to group by host and service"`
</ParamField>

<ParamField path="type" type="string" required>
  The type of alert grouping configuration. Must be one of:
  - `content_based` - Group by matching field values
  - `content_based_intelligent` - Intelligent grouping with field matching
  - `intelligent` - AI-powered intelligent grouping
  - `time` - Group alerts within a time window
  
  Example: `"content_based"`
</ParamField>

<ParamField path="config" type="object" required>
  The configuration for the alert grouping setting. Structure varies by type. See [Create Alert Grouping Setting](/tools/alert-grouping/create-setting) for detailed configuration options.
  
  Example for content-based:
  ```json
  {
    "aggregate": "all",
    "fields": ["host", "service"],
    "time_window": 1800
  }
  ```
</ParamField>

<ParamField path="services" type="array" required>
  Array of service references that this alert grouping setting applies to. Note: `content_based_intelligent` type allows only one service.
  
  Example:
  ```json
  [{"id": "PSVC123", "type": "service_reference"}]
  ```
</ParamField>

## Use Cases

- **Optimize grouping** - Adjust time windows or fields based on alert patterns
- **Fix configuration** - Correct misconfigured grouping rules
- **Adapt to changes** - Update grouping as alert structure evolves
- **Test improvements** - Experiment with different grouping strategies
- **Service changes** - Update which services use the grouping setting

## Update Examples

### Update Time Window
```json
{
  "setting_id": "PAGRSET1",
  "type": "content_based",
  "config": {
    "aggregate": "all",
    "fields": ["component"],
    "time_window": 1800
  },
  "services": [{"id": "PSVC123"}]
}
```

### Change to Intelligent Grouping
```json
{
  "setting_id": "PAGRSET1",
  "type": "intelligent",
  "config": {
    "time_window": 0,
    "iag_fields": ["summary", "description"]
  },
  "services": [{"id": "PSVC123"}]
}
```

### Update Grouping Fields
```json
{
  "setting_id": "PAGRSET1",
  "type": "content_based",
  "config": {
    "aggregate": "any",
    "fields": ["host", "service", "environment"],
    "time_window": 3600
  },
  "services": [{"id": "PSVC123"}]
}
```

## Important Notes

<Warning>
- Updates replace the entire configuration - include all required fields
- Changes take effect immediately for new alerts
- Existing incidents are not affected by the update
- The `content_based_intelligent` type supports only **one service**
- Setting `time_window` to 0 uses PagerDuty's recommended time window
- Ensure field names match your alert structure
</Warning>

<Note>
Before updating, consider reviewing the current configuration with [Get Alert Grouping Setting](/tools/alert-grouping/get-setting) to understand the existing setup.
</Note>
