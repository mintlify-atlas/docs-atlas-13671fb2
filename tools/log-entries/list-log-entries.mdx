## Overview

The `list_log_entries` tool retrieves all log entries across your PagerDuty account. Log entries are records of all events on your account, providing a comprehensive audit trail of actions taken by users, services, and integrations.

<Note>
This is a **read-only** tool that does not modify any data in your PagerDuty account. If no time range is specified, defaults to the last 7 days.
</Note>

## Parameters

<ParamField path="since" type="datetime" optional>
  The start of the date range to search. Defaults to 7 days ago if not specified.
  
  Example: `"2024-01-01T00:00:00Z"`
</ParamField>

<ParamField path="until" type="datetime" optional>
  The end of the date range to search. Defaults to now if not specified.
  
  Example: `"2024-01-31T23:59:59Z"`
</ParamField>

<ParamField path="limit" type="integer" optional default="100">
  Maximum number of results to return. Must be between 1 and 10000.
  
  Example: `50`
</ParamField>

<ParamField path="offset" type="integer" optional default="0">
  The offset for pagination. Used to retrieve additional pages of results.
  
  Example: `100`
</ParamField>

<ParamField path="is_overview" type="boolean" optional>
  If `true`, returns only the most important changes to incidents. Useful for getting a high-level view without excessive detail.
  
  Example: `true`
</ParamField>

<ParamField path="include" type="array" optional>
  Include additional related resources in the response. Available options:
  - `incidents` - Include incident details
  - `services` - Include service details
  - `channels` - Include channel information
  - `teams` - Include team information
  
  Example: `["incidents", "services"]`
</ParamField>

<ParamField path="team_ids" type="array" optional>
  Filter log entries by team IDs. Only returns log entries associated with the specified teams.
  
  Example: `["TEAM123", "TEAM456"]`
</ParamField>

<ParamField path="time_zone" type="string" optional>
  Time zone for the results. Uses standard timezone names.
  
  Example: `"America/New_York"` or `"UTC"`
</ParamField>

<ParamField path="total" type="boolean" optional>
  Set to `true` to include the total count of log entries in the response.
  
  Example: `true`
</ParamField>

## Response

Returns a list of log entry objects.

<ResponseField name="response" type="array">
  Array of log entry objects
  
  <Expandable title="LogEntry Object">
    <ResponseField name="id" type="string">
      The unique identifier of the log entry
    </ResponseField>
    
    <ResponseField name="type" type="string">
      The type of log entry (e.g., `trigger_log_entry`, `acknowledge_log_entry`, `resolve_log_entry`, `assign_log_entry`)
    </ResponseField>
    
    <ResponseField name="summary" type="string">
      A short description of the log entry
    </ResponseField>
    
    <ResponseField name="self" type="string">
      The API URL at which this log entry is accessible
    </ResponseField>
    
    <ResponseField name="html_url" type="string">
      The URL at which this log entry is accessible in the PagerDuty web UI
    </ResponseField>
    
    <ResponseField name="created_at" type="datetime">
      The date and time when this log entry was created
    </ResponseField>
    
    <ResponseField name="agent" type="object">
      The agent (user, service, or integration) that performed the action
      
      <Expandable title="Agent Object">
        <ResponseField name="id" type="string">
          The agent ID
        </ResponseField>
        
        <ResponseField name="type" type="string">
          The agent type (e.g., `user_reference`, `service_reference`)
        </ResponseField>
        
        <ResponseField name="summary" type="string">
          A short description of the agent
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="channel" type="object">
      The channel through which the action was performed (e.g., web, API, email)
    </ResponseField>
    
    <ResponseField name="service" type="object">
      Reference to the service associated with this log entry
    </ResponseField>
    
    <ResponseField name="incident" type="object">
      Reference to the incident associated with this log entry
    </ResponseField>
    
    <ResponseField name="teams" type="array">
      References to teams associated with this log entry
    </ResponseField>
  </Expandable>
</ResponseField>

## Use Cases

- **Audit trail** - Review all actions taken on your PagerDuty account

- **Compliance reporting** - Generate reports of who did what and when

- **User behavior analysis** - Understand how users interact with PagerDuty
- **Integration monitoring** - Track actions performed by integrations

## Important Notes

<Note>
- Log entries are read-only and cannot be modified or deleted
- The default time range is 7 days to ensure reasonable response times
- Use `is_overview: true` to reduce noise in high-traffic accounts
- Log entries include polymorphic types based on the action performed
</Note>
